# Rebuild Plan — lafayette on fresh Ubuntu 26.04 (2026-06-20)

Status-aware rebuild plan for the current root-SSD reinstall. Complements the
generic `docs/disaster-recovery.md` and the `bin/rebuild-lafayette` script.
**This is not from scratch** — the repo already ships a 9-stage guided rebuild
script. This doc records what's different *this time* and the right sequencing
given (a) the media drives aren't connected yet and (b) the OS jumped 20.04 →
26.04.

## 🔻 RESUME HERE — next steps (paused evening of 2026-06-20)

**Running overnight:** full `/home/will` restore from B2 (in `tmux` session
`restore`). First thing in the morning, confirm it finished cleanly.

Do these in order:

1. **Verify the home restore** — `tmux attach -t restore`, check restic exited 0,
   then `du -sh /home/will`. This also restored the missing build sources
   `~/code/homebot` and `~/code/parchment-server`.
2. **Restore appdata from B2** (if not already done) — gets gastown/guildmaster/
   health-server source + container data + `letsencrypt/acme.json`:
   ```bash
   cd /willflix
   sudo B2_ACCOUNT_ID="$(cat secrets/b2_account_id)" \
        B2_ACCOUNT_KEY="$(cat secrets/b2_application_key)" \
        RESTIC_PASSWORD_FILE="$PWD/secrets/restic_password" \
        restic -r b2:willflix-backup:lafayette restore latest --tag willflix \
          --target / --include /willflix/docker/appdata
   ```
   NOTE: snapshots are tagged by source (`home`, `willflix`, `postgres`, `plex`,
   `calibre`). Always pass `--tag <source>` — bare `restore latest` grabs the
   newest snapshot across *all* sources and silently restores 0 matching files.
   The `willflix` snapshot excludes postgres raw data (restored via pg_dump, step 7).
3. **Install runtimes (asdf)** — still pending; `node`/`go`/`python`/`uv`/`yarn`
   all missing. See A3 below. (node also revives the `ctx_*` MCP tooling.)
4. **Bring up Docker** — all build contexts now present, GHCR login wired in:
   ```bash
   docker network create --subnet 172.30.0.0/16 --gateway 172.30.0.1 config_default 2>/dev/null || true
   cd /willflix/docker && sudo docker compose -p config up -d
   ```
5. **Connect + mount the media drives** (Stage 5): attach drives, `blkid` to
   confirm labels, `sudo mount -a`, `mountpoint /Volumes/Media`.
6. **Restart media services** so they see real content:
   `sudo docker compose -p config restart` (Plex, *arr, qBittorrent).
7. **Postgres** (Stage 8) — once `Bonus1` is mounted, restore from
   `/Volumes/Bonus1/postgres-backup/latest` *unless* appdata already carried the
   DB data (check `docker exec config-postgres-1 psql -U will -l`).
8. **Verify** — `willflix-notify --test --subject "rebuild"` +
   `docker logs smtp-relay --tail 5`. Confirm ~38 containers, none unhealthy.
9. **Re-enable snapraid** (uncommented in repo `etc/root-crontab` on 2026-06-21 —
   new hardware, old PSU/hang cause gone). Once drives mount: `sudo snapraid sync`
   (baseline — root-SSD content file is fresh) → `sudo snapraid scrub` →
   `sudo crontab /willflix/etc/root-crontab` to activate the daily/weekly/freshness crons.
10. **Commit** the `bin/rebuild-lafayette` edits made 2026-06-20 (sources.list.d
    fix, optional-pull, GHCR login, dir-creation + build-context guard, restic
    added to packages, AI-rules topmatter).

⚠️ Until the drives are mounted, don't let qBittorrent/nzbget download — they'd
fill the root SSD via auto-created `/Volumes/Media*` dirs.

## The one blocker: git-crypt key

Nothing useful happens until secrets are decrypted — every Docker env var, the
B2 backup credentials, and `restic_password` all live in `secrets/**`.

```bash
# LastPass secure note: "lafayette git-crypt key" (base64)
echo '<paste-base64>' | base64 -d > /tmp/git-crypt-key
cd /willflix && git-crypt unlock /tmp/git-crypt-key && rm /tmp/git-crypt-key
# verify:
file secrets/smtp_password   # should say "ASCII text", not "data"
```

## ⚠️ Fix the script before running Stage 2

`bin/rebuild-lafayette:101` writes the Docker apt repo to
`/etc/apt/sources-list.d/docker.list` — **wrong dir** (`sources-list.d` doesn't
exist). The redirect fails, the repo is never added, and `apt-get install
docker-ce` then fails. Correct path is `/etc/apt/sources.list.d/docker.list`.
Fix this one character before running, or install Docker manually.

## Ubuntu 26.04 caveats (script was written for 22.04)

- Stage 1 now checks `VERSION_ID="26.04"` — it only warns + asks to continue, so a mismatch is fine, but expect the prompt.
- Docker's apt repo uses `$VERSION_CODENAME` dynamically. If Docker hasn't
  published a 26.04 (codename) component yet, pin to the latest published LTS
  codename in the repo line, or use `apt install docker.io docker-compose-v2`
  from Ubuntu's own repo as a fallback.
- `mergerfs` / `snapraid` / `smartmontools` should all be in 26.04's archive;
  versions will differ from 20.04 but configs are compatible.

## 2026-06-24 — all `*.willflix.org` down a week after rebuild (gaps now fixed in the script)

Every SSO host 404'd, then 502'd. All causes were rebuild fallout; the script now
handles each, but watch for them:

1. **Traefik was down and too old for the 26.04 daemon.** Docker on 26.04 requires
   daemon API ≥ 1.40; Traefik **v2.10 hardcodes API 1.24** and *ignores*
   `DOCKER_API_VERSION` → its docker provider can't connect → zero label-based
   routers → 404 on every service (file-provider routes like code.willflix.org
   still worked, which is the tell). Fixed in compose: **`traefik:v2.11`**
   (adds API-version negotiation) + `restart: unless-stopped`.
2. **`docker/secrets` symlink missing** → every secret-using container failed to
   start. Stage 6 now creates `docker/secrets → ../secrets`.
3. **`config_default` network** must be `172.30.0.0/16` (gateway `172.30.0.1` is how
   Traefik reaches host services such as opencode `:4097`). Stage 7 creates it with
   that subnet — never `docker network create config_default` bare (it gets a
   random subnet and breaks host-service routing).
4. **Postgres came up empty.** Phase A tells you to skip Stage 8; if Phase B's
   Stage 8 is then skipped or fails, you get no databases and Authentik
   crash-loops (`role "authentik" does not exist`) → 502 on every SSO host. The
   backup at `/Volumes/Bonus1/postgres-backup/latest/` is **complete** (globals +
   authentik/nextcloud/openbrain/healthdata). Restore order: globals (roles) →
   `createdb` → load each dump.
5. **Compose project name.** The live stack and CLAUDE.md use project **`docker`**
   (bare `docker compose` in `/willflix/docker`), not `config`. `COMPOSE_PROJECT`
   and Stage 8's container refs are corrected to `docker` so a rebuild doesn't
   split-brain against the running stack.

For manual recovery on an already-running box (not a full rebuild), see the agent
memory `willflix-rebuild-postgres-proxy-gaps`.
